# Decomposition Rules (v0.1)

- Decompose only after PRD is stable. If PRD is not approved, warn and ask whether to continue.
- Prefer bounded-context tasks: one primary change area per task.
- Every task must reference at least one of:
  - a Journey (J#)
  - a Functional Requirement (FR-#)
  - an Acceptance Criterion (AC-#)
- Each milestone should have a clear deliverable and verification step.
